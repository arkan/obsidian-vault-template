# Vault Map

Generate a static, agent-readable index of all vault entities and collections. This file serves as the primary navigation tool for Claude Code — a machine-readable alternative to Obsidian's Dataview-powered MOCs.

## Trigger

`/vault-map` — regenerate the vault map

## Output

Write to `.claude/vault-map.md` (overwrite each time).

## Workflow

### Step 1 — Scan entities

Extract frontmatter from entity files using grep (do NOT read full files):

**Projects** — scan `Projects/*.md`:
- Extract: `type`, `status`, `area`, `last_synthesis`
- Only include files with `type: project`

**People** — scan `Areas/People/*.md`:
- Extract: `type`, `status`, `fact_count`, `last_synthesis`
- Only include files with `type: person`

**Companies** — scan `Areas/Companies/*.md`:
- Extract: `type`, `status`, `fact_count`, `last_synthesis`
- Only include files with `type: company`

### Step 2 — Count collections

Count `.md` files (excluding MOCs and non-note files) in:
- `Resources/GitHub-Stars/` — github stars
- `Resources/X-Bookmarks/` — X bookmarks
- `Resources/ImportedURLs/` — imported URLs
- `Meetings/` — meetings
- `Daily Notes/` (recursive) — daily notes

### Step 3 — Read recent activity

Read the last 10 data rows from `Areas/Claude/operations-log.md` (if it exists).

### Step 4 — Write vault-map

Write `.claude/vault-map.md` with this format:

```markdown
# Vault Map

> Auto-generated by `/vault-map`. Do not edit manually.
> Last updated: YYYY-MM-DD HH:MM

## Projects (N)

| Name | Status | Area | Last synthesis |
|------|--------|------|----------------|
| [[Project-Name]] | active | [[Area]] | YYYY-MM-DD |
...

## People (N)

| Name | Status | Facts | Last synthesis |
|------|--------|-------|----------------|
| [[Prenom Nom]] | active | 5 | YYYY-MM-DD |
...

## Companies (N)

| Name | Status | Facts | Last synthesis |
|------|--------|-------|----------------|
| [[Company-Name]] | active | 3 | YYYY-MM-DD |
...

## Collections

| Collection | Count | Location |
|------------|-------|----------|
| GitHub Stars | N | Resources/GitHub-Stars/ |
| X Bookmarks | N | Resources/X-Bookmarks/ |
| Imported URLs | N | Resources/ImportedURLs/ |
| Meetings | N | Meetings/ |
| Daily Notes | N | Daily Notes/ |

## Recent Activity

| Date | Op | Skill | Result | Scope |
|------|----|-------|--------|-------|
(last 10 rows from operations-log.md)
```

If a section has 0 items, show the table header with "(aucun)" in the first row.

### Step 5 — Log

Append one line to `Areas/Claude/operations-log.md`:
```
| YYYY-MM-DD HH:MM | generate | vault-map | régénéré | .claude/vault-map.md |
```

### Step 6 — Confirm

Report: "Vault map régénéré : X projets, Y personnes, Z entreprises, N collections."

## Rules

- Use grep to extract frontmatter — do NOT read full files
- Entity names come from filenames (strip `.md`), wrapped in `[[wikilinks]]`
- Sort entities alphabetically within each section
- The vault-map is a snapshot — it gets stale. Regenerate during evening review or on demand.
- If `Areas/Claude/operations-log.md` doesn't exist, omit the Recent Activity section
