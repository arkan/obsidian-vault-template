# AGENTS.md

> **Template vault** for Codex + Obsidian. Adapt this file to your personal use: replace fictional examples with your real projects, goals, and entities.

This file provides guidance to Codex agents when working with code in this repository.

## What This Is

Obsidian "second-brain" vault. PARA organization (Projects, Areas, Resources, Archive). Auto-synced to git via obsidian-git plugin (auto-commit/push/pull every 10 min).

## Note Conventions

### Frontmatter

All notes use YAML frontmatter with `type` field. Available types:
- `project`, `person`, `company`, `note`, `dashboard`, `moc`, `meeting`, `review`, `daily`, `codex`, `goals`, `infrastructure`, `article`

### Statuses

`status` field for entities (project/person/company):
- `backlog` — identified but not yet started
- `active` — currently being worked on
- `waiting` — blocked by a third party
- `review` — awaiting validation
- `done` — completed (before archiving)
- `archived` — moved to Archive/
- `inactive` — for people/companies no longer actively engaged

For articles (`type: article`):
- `idea` — captured idea
- `outline` — structured outline
- `draft` — draft in progress
- `review` — under review
- `published` — published

### Entity Templates (project/person/company)

Common fields: `type`, `aliases`, `tags`, `status`, `created`, `last_synthesis`, `fact_count`
Additional project field: `area` — link to the responsible area (e.g., `"[[Infrastructure]]"`)
Body sections: `## Active Facts` then `## Superseded Facts`

### Daily Notes

Fields: `type: daily`, `date`, `mood`, `sleep`. Sections: `## Tasks`, `## Notes`, `## Log`.
Daily notes are created lazily (via `/review` or other skills), not pre-generated.

### Daily Codex Summaries

Stored in `Areas/Codex/YYYY-MM-DD-codex.md` (one file per day). Generated by the `daily-codex-resume` skill. Sections: `## Summary`, `## Activity by Project`, `## Statistics`, `## Connections`.

### Tasks (TODO.md)

`Areas/TODO.md` uses the **obsidian-tasks** format:
```
- [ ] Description #project/xxx ⏫ 📅 2026-04-01
```
Priorities: `⏫` (high), `🔼` (medium), `🔽` (low). Completion: `✅ YYYY-MM-DD`.
Sections: `## Tasks` (active) then `## Archive` (completed).

### Facts (Entity Knowledge)

Entities store facts under `## Active Facts` and `## Superseded Facts`.
Format: `` - `<prefix>-NNN` Fact description #status/active ``
IDs: prefix = filename in lowercase with dashes, auto-incremented number (e.g., `saas-dashboard-001`, `mobile-app-002`).

### Linking

Use `[[wikilinks]]` for Obsidian graph connectivity. When creating Codex activity summaries, link known people, projects, and services inline.

## Classification Strategy

- **Folders** = primary classification (PARA: Projects, Areas, Resources, Archive)
- **Tags** = cross-cutting classification (`#tech`, `#client/xxx`, `#urgent`, `#meeting`)
- **Wikilinks** = relationships between entities (`[[Person Name]]`, `[[Project]]`)
- Do not duplicate information across the three systems

## File Naming Conventions

- **Projects**: `Project-Name.md` (PascalCase with dashes)
- **People**: `First Last.md`
- **Companies**: `Company-Name.md`
- **Daily notes**: `YYYY-MM-DD.md` (automatic)
- **Meetings**: `YYYY-MM-DD Subject.md`
- **MOCs**: `MOC Name.md`

## Plugins

Active community plugins: calendar, obsidian-git, excalidraw, templater-obsidian, dataview, obsidian-tasks-plugin, homepage.

- **Templater**: uses `<% tp.date.now() %>` and `<% tp.file.title %>` syntax in templates
- **Dataview**: powers Home.md and MOC queries — do not break `dataview` code blocks
- **Daily Notes**: format `YYYY/YYYY-MM/YYYY-MM-DD`, folder `/Daily Notes/`, template `/Templates/daily-note.md`

## Important Rules

- Content language is English — all note text, section headers, and descriptions in English
- Never modify `.obsidian/` plugin configs unless explicitly asked
- Respect existing frontmatter schema when creating/editing notes
- The vault auto-commits; avoid creating merge conflicts with rapid file changes
- Inbox is the single entry point for quick capture — sort during weekly review or evening review
- MOCs are the navigation layer — Home.md points to MOCs, not directly to all entities
- The `.agents/vault-map.md` is the machine-readable index for agents. Regenerate it via `/vault-map` after significant vault changes. The `Areas/Codex/operations-log.md` is append-only — every skill that mutates vault content should append a line.
