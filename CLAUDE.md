## Note Conventions

### Frontmatter

All notes use YAML frontmatter with `type` field. Available types:
- `project`, `person`, `company`, `note`, `dashboard`, `moc`, `meeting`, `review`, `daily`, `claude`, `goals`, `infrastructure`, `article`

### Statuses

`status` field for entities (project/person/company):
- `backlog` â€” identified but not yet started
- `active` â€” currently being worked on
- `waiting` â€” blocked by a third party
- `review` â€” awaiting validation
- `done` â€” completed (before archiving)
- `archived` â€” moved to Archive/
- `inactive` â€” for people/companies no longer actively engaged

For articles (`type: article`):
- `idea` â€” captured idea
- `outline` â€” structured outline
- `draft` â€” draft in progress
- `review` â€” under review
- `published` â€” published

### Entity Templates (project/person/company)

Common fields: `type`, `aliases`, `tags`, `status`, `created`, `last_synthesis`, `fact_count`
Additional project field: `area` â€” link to the responsible area (e.g., `"[[Infrastructure]]"`)
Body sections: `## Active Facts` then `## Superseded Facts`

### Daily Notes

Fields: `type: daily`, `date`, `mood`, `sleep`. Sections: `## Tasks`, `## Notes`, `## Log`.
Daily notes are created lazily (via `/review` or other skills), not pre-generated.

### Daily Claude Summaries

Stored in `Areas/Claude/YYYY-MM-DD-claude.md` (one file per day). Generated by the `daily-claude-code-resume` skill. Sections: `## Summary`, `## Activity by Project`, `## Statistics`, `## Connections`.

### Facts (Entity Knowledge)

Entities store facts under `## Active Facts` and `## Superseded Facts`.

### Linking

Use `[[wikilinks]]` for Obsidian graph connectivity. When creating Claude activity summaries, link known people, projects, and services inline.

## Classification Strategy

- **Folders** = primary classification (PARA: Projects, Areas, Resources, Archive)
- **Tags** = cross-cutting classification (`#tech`, `#client/xxx`, `#urgent`, `#meeting`)
- **Wikilinks** = relationships between entities (`[[Person Name]]`, `[[Project]]`)
- Do not duplicate information across the three systems
